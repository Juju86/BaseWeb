Grid System

Grid System

BaseWeb's grid system is a class based and semantic mixin based system. You can output grid styles using the class based mixins or apply the grid mixins directoy to page elements for a more semantic system.

Class Based Grid System

The class based system works by adding the required markup and classes to a page. It works by creating a container element, rows and then columns. Using the default grid system settings the markup looks something like this:

<div class="container">
  <div class="row">
    <div class="col col-4">
      ...
    </div>
    <div class="col col-8">
      ...
    </div>
  </div>
  <div class="row">
    <div class="col col-10">
      ...
    </div>
    <div class="col col-2">
      ...
    </div>
  </div>
</div>

The grid classes by default are only applied from tablet and above media break points, but you can adjust the output as needed. By default, the mobile grid system is output using .build-mobile-grid-system(); and then the default class system is used for tablets and above using the default values in your variables file and the .build-grid-system(); mixin.

Default Grid Variables

Variable Description Default Value
@grid-total-width The total width that the grid system takes. Setting this value to 100% makes it a fluid grid. 100%
@grid-gutter-width The gutter width (space between columns). If the grid system is fixed, the value should be in pixels and percentage if it's fluid. Notes: this space is created using the margin-left property. 2%
@grid-columns Number of columns the grid system should have. 12

Semantic Grid System

The semantic grid system can be used by calling the respective make-* mixins to existing markup. Although, you still need the same amount of elements to create the grids, they can have any class you'd like to target. So for example, you may have some markup like this:

<div class="wrapper">
  <div class="content">
    <div class="aside">
      ... sidebar content goes here ...
    </div>
    <div class="article">
      ... article content goes here ...
    </div>
  </div>
</div>

Using the mixins to create the grid with this markup would look like this:

.wrapper {
  .make-container();
}
.content {
  .make-row();
}
.aside {
  .make-column(4);
}
.article {
  .make-column(8);
}

And the result looks like this:

... sidebar content goes here ...
... article content goes here ...

You can pass in custom paramaters for these mixins as well, but by default they use the values set in your variables file. The only required perameters are the amount of columns you want to span when using the .make-column(*); mixin.

Media Queries

The default media queries that BaseWeb uses are mobile-first. What that means is that the default styles that you write will be overridden by subequent queries starting from mobile and ending with large screen styles. With that in mind, if you need support for browsers that don't support media queries, you'll have to update responsive.less so that the default queries are desktop first, and then the mobile styles would override them.

The default media queries look like this:

/**
 * Mobile First
 * The output of #responsive > .mobile();
 */
#responsive > .mobile();

/**
 * Small
 * The output of #responsive > .small();
 */
@media screen and (max-width: @bp-small ) {
  #responsive > .small();
}

/**
 * Tablet
 * The output of #responsive > .tablet();
 */
@media screen and ( min-width: @bp-tablet ) {
  #responsive > .tablet();
}

/**
 * Desktop
 * The output of #responsive > .desktop();
 */
@media screen and ( min-width: @bp-desktop ) {
  #responsive > .desktop();
}

/**
 * Large
 * The output of #responsive > .large();
 */
@media screen and ( min-width: @bp-large ) {
  #responsive > .large();
}

The breakpoints for media queries are set in your variables file and can be changed to whatever you like.

Media Query Templates

When writing styles, BaseWeb provides you with a simple way to add styles to a media query without having to write the queries themselves. You can do this using the media query templates who's syntax looks like this:

#responsive {
  .small()   {}
  .mobile()  {}
  .tablet()  {}
  .desktop() {}
  .large()   {}
}

So, in your custom.less styles, you can now write mobile tablet specific styles like this:

#responsive {
  .tablet()  {
    // Your tablet specific styles go here
  }
}

Essentially what this does is store your styles in the #responsive { .tablet() {} } mixin which gets called in responsive.less in their respective media queries.
